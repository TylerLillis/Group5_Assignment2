Project Idea

Overview

Our project idea is an interactive program that informs and tests the knowledge of the users on basic cyber security practices and everyday security practices that should be followed on all internet of things devices. The aim is to build this program in Python and explain all of the necessary information in an easy to understand and coherent manner. Specifically, we want the program to be modular, to allow the easy addition of information as required, and to include basic text information on these security practices as well as an interactive section that will quiz the user on the information it has previously explained in the aim for the user to retain that knowledge. We believe that a program, rather than a website has the flexibility and the modularity that is required to properly educate users in an interactive digital environment.

Motivation

This project is useful because it will be aimed at teaching in an easy-to-understand manner to allow almost any level of computer knowledge to be able to use and understand the information contained in the program. “Attacks on the Internet of Things devices will increase rapidly due to hypergrowth in the number of connected objects, poor security hygiene, and the high value of data on IoT devices”(Sciencedirect.com, 2015, page 26 para. 2), this rise in attacks really exemplifies the need to educate people who aren’t as knowledgeable about tech and inform them how they can protect themselves. Especially due to the increasing reliance on devices to perform everyday activities. we decided to pick this project specifically as it strongly connects to two of our member’s ideal jobs as a Cybersecurity Specialist and a Cyber Security Analyst through informing others on how they can improve their system’s security and will require us to thoroughly research these practices to be able to explain them in a clear and easy-to-understand manner.

Tools and Technologies

Python - For this program, we plan to write it in Python. This is due to us already being somewhat familiar with it while also being a language that can run on Windows, macOS and Linux enabling anyone running these operating systems to use the program. (Might consider TempleOS for version 2.0)

PyQt5 – This is the python UI module we plan to use for the UI of the program. We picked this because while it does have a steeper learning curve than other UI modules, it is much more robust and highly customizable to better suit our needs. Whereas a module such as Tkinter is too basic and harder to make it suit our needs. A designer application has also been developed for PyQt5 which would allow us to create the UI relatively easily without having to manually code each aspect of the interface (which would be really time consuming).

XML.etree.ElementTree – Python module that will be used to parse data from the XML file where descriptions, questions and answers will be stored.

PC – Any computer would be required to run the program

Description

We have decided to include the following features in our program:

Teaching/Testing sections
We want the two major features of the program to be to explain various security practices to the user and to test their knowledge afterwards as a method of retaining the information. For the teaching section, we will aim to include short and simple explanations alongside diagrams and other visual imagery to improve understanding, all the teaching sections will be categorized under online safety, privacy, social media/communication safety, device safety, and password safety. Online safety will focus on installing adblockers and their importance, safe use of online shopping and trusted online marketplaces and basic scams to look out for and avoid. Privacy will focus on how you can keep your personal information safe by splitting accounts and services across different emails, using different aliases to avoid being profiled online, and important settings to look for within services that keep the information you give to a minimum. Social media/communication safety will focus on identifying suspicious or phishing emails and to avoid blindly trusting strangers online. Device safety will focus on the importance of an antivirus encryption and why it is necessary and creating backups of important documents. Password safety will focus on the use of password managers and why they are important. A potential limitation of this section is the possibility of giving too much information too quickly or just confusing the user. To mitigate this problem, word choice and diagrams need to be carefully created to be simple enough to understand, but still correct. For the testing section, the aim is to keep a record of which sections the user has been through and give them multiple-choice questions related to those topics, the questions will be pre-written and randomized in the order they are given to the user to disallow them remembering the order of the answers over the information.

Modularity
To avoid having to hard code in each page and question into the program, we aim to make the program modular by instead reading the data that contains descriptions, questions, and answers from a storage format such as an XML file. This will make it quite easy to update or make changes to the content in the program. Images can be stored in a separate folder. A potential drawback is it may be hard to indicate where a certain image or diagram is placed, and it may overcomplicate the creation of the program beyond our abilities.

Focus on the UI
Having a clean, easy to use UI will be essential for this program. This is due to the wide range of abilities with computers that the users will have, we do not want to confuse or turn away potential users. In the creation of the UI, we will also have to take accessibility into account to accommodate for more people. Eg, larger text to accommodate for the degradation of an elderly person’s sight. We plan to minimize the amount of UI elements shown on the screen to avoid making the users lost or confused on where to click, using accent colours to identify key components that clearly contrast with the background and reduce distractions.

For the UI in this project, our team has decided to implement Google’s Material Design 2 system theme as it is a well-known standard among applications and it is simplistic in design. Which is suitable for our planned userbase. We decided to avoid other popular UI trends such as Neumorphism and Glassmorphism as they are not as well established or known as Material Design so they would provide our users with a consistent experience with something that they are more likely to recognize. Also, material design is under the Apache-2.0 license so it is free to use without any legal issues.

Computers over mobile devices
We plan to make this a program that can be run on a PC over a phone application as most of the information we plan to explain to the user can be applied most effectively on a PC. Whereas mobile devices are currently a lot safer than PCs due to being more locked down (eg. Hard to install 3rd party apps on iOS, need to turn on a setting for Android) and security features are being integrated into newer systems. In addition to this, the program will be installed via executable, a feature that may assist users who are unfamiliar with the practice of extracting files. Some drawbacks we can identify due to this decision is the inability for our program to reach anyone who does not own a computer and the requirement to install this program beforehand may confuse potential users.

Reward System
To maintain the user’s attention to the program, we will be introducing gamification into the project, which is linked to Chelsie’s project idea, by incorporating an achievement-based reward system. Since the program revolves around teaching and informing users on cybersecurity practices, it may be difficult to keep users engaged as they progress through the teaching sections. Adding a feature that allows for achievements—trophies at the end of every section can give those learning cybersecurity practices a sense of accomplishment after learning valuable information.

Skills Required

The skills required for this project would mainly include knowledge in the python programming language, extending to writing UI code through the PyQt5 module along with parsing data through an XML module. These skills could be acquired through video tutorials found on YouTube, or digital learning platforms such as w3schools.com. Another avenue for gaining these skills is joining online programming communities, such as stackoverflow.com, which allows users to ask and answer programming-based questions. Also, knowledge of good UI practices would be necessary to create a fluent and understandable user interface. This can be done through the reading of books by notable experts in the field such as Dr. Jacob Nielson, and other contemporaries, whose writings outline the fundamental principles of user design.  Knowledge of good cyber security practices would be essential before being able to explain the appropriate knowledge to the user without flaws. This knowledge could be achieved through various readings and analysis of peer-reviewed journals and other academic articles relating to the field of cyber-security. These skills are reasonably feasible and would take on estimate two or three months to learn from scratch. This is due to the bulk of the learning-based around understanding python and its modules.

Outcome

The intended outcome for this project is to inform as many potential victims of cybercrime as possible. Realistically, this project won’t create a massive impact on those affected by cyberattacks but the more people who have this knowledge and understand these concepts, the better. The original problem would be solved by teaching vulnerable people how they can keep themselves safe online and hopefully leading to them applying these practices or avoiding getting scammed/attacked due to the information that the program covers.
